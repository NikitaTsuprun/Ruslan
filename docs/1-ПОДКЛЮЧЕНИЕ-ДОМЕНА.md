# Подключение домена yuristpro1.ru к сайту

Сайт собирается статически (Nuxt → `nuxt generate`) и публикуется на **GitHub Pages**.
Чтобы он открывался по адресу **https://yuristpro1.ru**, нужно сделать два дела:
**(1) в коде** — уже сделано мной, **(2) у регистратора домена и в настройках GitHub** — делаешь ты по шагам ниже.

---

## ✅ Что уже сделано в коде (мной)

1. Добавлен файл `public/CNAME` со значением `yuristpro1.ru` — он привязывает домен к GitHub Pages при каждом деплое.
2. В сборке переключён базовый путь с `/Ruslan/` на `/` (файл `.github/workflows/deploy.yml`, переменная `NUXT_APP_BASE_URL: /`). Теперь все ссылки, картинки, `robots.txt` и `sitemap.xml` работают от корня домена.
3. В коде уже прописан правильный адрес сайта `https://yuristpro1.ru` (canonical, sitemap, Open Graph, микроразметка).

> ⚠️ **Важно про порядок.** После переключения на `base = /` старый адрес `…github.io/Ruslan/` временно перестанет работать (стили «уедут»). Это нормально: сайт «переезжает» на домен. Поэтому сначала настрой DNS и домен в GitHub (шаги ниже), и только потом деплой.

---

## Шаг 1. DNS-записи у регистратора домена

Зайди в панель управления доменом `yuristpro1.ru` (там, где покупал — reg.ru, timeweb, nic.ru и т.п.), раздел **DNS-записи / Управление зоной**.

### 1.1. Записи типа `A` (для корневого домена `yuristpro1.ru`)

Добавь **четыре** записи `A` с именем `@` (или пустым — это и есть корень домена), указывающие на серверы GitHub Pages:

| Тип | Имя (Host) | Значение (Value) |
|-----|-----------|------------------|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

### 1.2. Записи типа `AAAA` (IPv6, желательно — для надёжности)

| Тип | Имя (Host) | Значение (Value) |
|-----|-----------|------------------|
| AAAA | @ | 2606:50c0:8000::153 |
| AAAA | @ | 2606:50c0:8001::153 |
| AAAA | @ | 2606:50c0:8002::153 |
| AAAA | @ | 2606:50c0:8003::153 |

### 1.3. Запись `CNAME` для `www` (чтобы `www.yuristpro1.ru` тоже открывался)

| Тип | Имя (Host) | Значение (Value) |
|-----|-----------|------------------|
| CNAME | www | **ИМЯ-АККАУНТА.github.io.** |

Вместо `ИМЯ-АККАУНТА` подставь логин GitHub-аккаунта, где лежит репозиторий с включённым Pages (тот, чей адрес был `…github.io/Ruslan/`). Например, если это `nikitatsuprun`, то значение — `nikitatsuprun.github.io.` (точка в конце — если регистратор её требует).

> Если раньше на домене уже висели какие-то `A`/`AAAA`/`CNAME`-записи для `@` и `www` — **удали старые**, оставь только эти. Иначе будет конфликт.

DNS обновляется не мгновенно: обычно 15 минут – 2 часа, иногда до 24 часов.

---

## Шаг 2. Указать домен в настройках GitHub Pages

1. Открой репозиторий на GitHub → **Settings** → слева **Pages**.
2. В блоке **Custom domain** впиши `yuristpro1.ru` → **Save**.
   *(файл `CNAME` в коде тоже это делает автоматически при деплое, но лучше задать и вручную).*
3. GitHub начнёт проверку DNS (**DNS check in progress**). Когда записи из шага 1 подтянутся — появится зелёная галочка.
4. После успешной проверки поставь галочку **Enforce HTTPS** (принудительный HTTPS). Она станет доступна, когда GitHub выпустит бесплатный SSL-сертификат (может занять до часа после подтверждения DNS).

---

## Шаг 3. Деплой

После того как в GitHub Pages домен подтверждён:

1. Закоммить изменения кода (в GitHub Desktop: **Commit to main** → **Push origin**).
2. Пуш автоматически запустит GitHub Actions (вкладка **Actions** в репозитории) — сборку и публикацию.
3. Через 1–2 минуты открой **https://yuristpro1.ru** — сайт должен открыться со стилями и картинками.

---

## Шаг 4. Проверить, что всё встало правильно

- **https://yuristpro1.ru** — открывается, замок HTTPS зелёный.
- **https://www.yuristpro1.ru** — открывается и переадресует на без-www (или просто открывается).
- **https://yuristpro1.ru/robots.txt** — отдаёт текст (со ссылкой на sitemap).
- **https://yuristpro1.ru/sitemap.xml** — отдаёт список страниц.
- **https://yuristpro1.ru/keysy** — страница кейсов открывается.

## Частые проблемы

| Симптом | Причина / решение |
|--------|-------------------|
| «Сайт не открывается» сразу после настройки | DNS ещё не обновился. Подожди до 2–24 ч. Проверить можно на dnschecker.org (ввести `yuristpro1.ru`, тип A → должны быть адреса 185.199.108–111.153). |
| Открывается, но без стилей/картинок | Деплой прошёл со старым `base=/Ruslan/`. Убедись, что в `.github/workflows/deploy.yml` стоит `NUXT_APP_BASE_URL: /`, и пересобери (Actions → Re-run). |
| В GitHub Pages «Custom domain … is not properly configured» | Не хватает `A`/`AAAA`-записей или есть лишние старые записи. Перепроверь шаг 1. |
| Нет галочки «Enforce HTTPS» | Сертификат ещё выпускается. Подожди до часа после подтверждения DNS, обнови страницу настроек. |
| Домен уже привязан к другому репозиторию | Один домен = одни GitHub Pages. Убери домен из чужого репозитория, добавь в нужный. |

> После подключения домена не забудь в **Яндекс.Вебмастере** и **Яндекс.Метрике** тоже указать новый адрес `https://yuristpro1.ru` (см. `3-ИНСТРУКЦИЯ-ЯНДЕКС.md`).
