# Как пользоваться Яндексом: Метрика, Вебмастер, Директ

Простая пошаговая инструкция для сайта **yuristpro1.ru**. Три сервиса Яндекса и зачем каждый:

- **Яндекс.Метрика** — считает посетителей и заявки (конверсии). Мозг всей аналитики. Уже стоит на сайте.
- **Яндекс.Вебмастер** — показывает, как сайт индексируется в поиске Яндекса. Для бесплатного (SEO) трафика.
- **Яндекс.Директ** — платная реклама. Приводит клиентов за деньги. Настройка — в `2-ЯНДЕКС-ДИРЕКТ-ПАКЕТ.md`.

Все три входят под одним логином Яндекса. Управлять удобно из **Яндекс Бизнес / рекламного кабинета**.

---

## Часть 1. Яндекс.Метрика (обязательно настроить первой)

Счётчик Метрики **№ 110042390** уже встроен в сайт. Но чтобы считались заявки и звонки (а не только посещения), нужно **завести цели**. Без целей Директ не сможет оптимизироваться.

### 1.1. Как войти
1. Зайдите на **metrika.yandex.ru** под логином, к которому привязан счётчик 110042390.
2. Откройте счётчик «Разблокировка карт и счетов…».

### 1.2. Завести цели (главное!)

В коде сайта уже отправляются 5 событий. Нужно создать под них цели в Метрике.

Идите: **Счётчик → Настройка → вкладка «Цели» → «Добавить цель»**. Для каждой цели:
- **Тип условия:** «JavaScript-событие».
- **Идентификатор:** впишите точное название из таблицы (латиницей, без пробелов).
- **Название:** любое понятное (для себя).

| Идентификатор (вписать точно) | Название цели | Что засчитывает |
|---|---|---|
| `lead` | Заявка с формы | **Главная конверсия** — отправлена форма |
| `phone_click` | Клик по телефону | Нажали на номер телефона |
| `messenger_click` | Клик по мессенджеру | Нажали Telegram / WhatsApp / MAX |
| `social_click` | Клик по соцсети | Нажали ВК / Авито / TikTok / Instagram |
| `cta_click` | Клик по кнопке заявки | Нажали «Оставить заявку/Получить консультацию» |

> `lead`, `phone_click`, `messenger_click` — самые важные, их Директ использует для оптимизации. `cta_click` — вспомогательная (для ретаргетинга).

### 1.3. Проверить, что цели работают
1. Откройте сайт **yuristpro1.ru** в браузере.
2. Нажмите на телефон, потом на мессенджер, потом отправьте тестовую заявку через форму.
3. В Метрике: **Отчёты → Конверсии** — через несколько минут там должны появиться достижения целей. Либо в реальном времени: **Отчёты → Посещаемость / Вебвизор** (вебвизор в коде выключен для скорости — при желании включается в настройках счётчика).

### 1.4. Куда смотреть каждый день/неделю
- **Сводка** — посетители, источники, устройства.
- **Отчёты → Конверсии** — сколько заявок/звонков и откуда (это ваш главный отчёт).
- **Отчёты → Источники → Сводка** — откуда приходят люди (поиск, Директ, соцсети).
- **Карта кликов / Вебвизор** (если включите) — как ведут себя на сайте.

## Часть 2. Яндекс.Вебмастер (бесплатный трафик из поиска)

Сайт уже подтверждён в Вебмастере (файл `yandex_07d7828b142d5842.html` и мета-тег в коде). Но при переезде на домен нужно кое-что сделать.

### 2.1. После подключения домена yuristpro1.ru
1. Зайдите на **webmaster.yandex.ru**.
2. Добавьте сайт `https://yuristpro1.ru` (если раньше был добавлен адрес github.io — добавьте домен как новый сайт; права подтвердятся автоматически, т.к. файл верификации уже в коде).
3. **Индексирование → Файлы Sitemap** → добавьте `https://yuristpro1.ru/sitemap.xml`. Он уже генерируется автоматически (главная, кейсы, политика — с датами обновления).
4. **Индексирование → Переобход страниц** → отправьте на переобход `/`, `/keysy/`, `/politika-konfidencialnosti/`.
5. Проверьте **robots.txt**: `https://yuristpro1.ru/robots.txt` должен открываться (сайт открыт для индексации).

### 2.2. Что смотреть в Вебмастере (раз в 1–2 недели)
- **Индексирование → Страницы в поиске** — сколько страниц в индексе, нет ли выпавших.
- **Диагностика → Диагностика сайта** — ошибки и предупреждения (исправлять по списку).
- **Поисковые запросы** — по каким запросам вас находят и показывают.
- **Информация о сайте → Индекс качества (ИКС)** — общий «вес» сайта.
- **Ссылки** — кто ссылается на сайт.

### 2.3. Разово полезно
- Заполнить **«Информация о сайте»** (регион — Россия, тематика — юридические услуги).
- Подключить **Яндекс Бизнес** (карточка организации) — усиливает доверие и добавляет сайт в Яндекс.Карты/Услуги. Особенно полезно для юруслуг.

## Часть 3. Яндекс.Директ (платная реклама)

Полный готовый пакет (кампании, ключи, минус-слова, объявления) — в файле **`2-ЯНДЕКС-ДИРЕКТ-ПАКЕТ.md`**. Здесь — как этим пользоваться в интерфейсе.

### 3.1. Связать Директ с Метрикой (обязательно)
1. Зайдите на **direct.yandex.ru** под тем же логином, что и Метрика.
2. При создании/в настройках кампании в разделе «Метрика» укажите счётчик **110042390**.
3. В **«Ключевые цели»** кампании выберите цель `lead` (и при желании `phone_click`, `messenger_click`) — по ним Директ будет оптимизировать показы.

### 3.2. Запуск первой кампании
Действуйте строго по разделам 1–2 файла `2-ЯНДЕКС-ДИРЕКТ-ПАКЕТ.md`:
1. Создайте кампанию **«Поиск»**, гео — вся Россия.
2. Вставьте ключевые фразы группы 🔥 «горячие» (раздел 4 пакета).
3. Вставьте минус-слова (раздел 5 пакета).
4. Создайте объявления (раздел 6), заполните быстрые ссылки, уточнения и визитку (раздел 7).
5. Поставьте ручную стратегию с потолком цены клика и недельный бюджет (при 300–500 ₽/день ≈ 2100–3500 ₽/нед).
6. Настройте расписание и корректировки ставок (раздел 8 пакета).
7. Отправьте на модерацию. Юрниша строгая — **без обещаний «100% результата»**.

### 3.3. Ежедневная рутина первые 2 недели (5 минут)
1. **Директ → отчёт «Поисковые запросы»** — смотрите, по каким РЕАЛЬНЫМ запросам показывались объявления.
2. Всё нецелевое (инфо-запросы, «бесплатно», «самому», другие темы) → добавляйте в **минус-слова**. Это главная экономия бюджета.
3. Отключайте фразы, которые тратят деньги без заявок.

### 3.4. Раз в неделю
- **Мастер отчётов** в Директе: смотрите расход, клики, **конверсии** (цель `lead`), цену заявки.
- Ставки: поднимайте на рабочих фразах, снижайте/отключайте на дорогих без конверсий.
- Когда наберётся **10+ конверсий** — переключите стратегию на **«Оплата за конверсии»** (Директ сам будет приводить заявки по целевой цене).

### 3.5. Когда расширяться
Только после того, как первая кампания стабильно даёт заявки:
- добавить кампанию по 161-ФЗ / базе ЦБ,
- добавить кампанию по ИП/ООО (заявки дороже, но ценнее),
- подключить РСЯ-ретаргетинг на тех, кто заходил и не оставил заявку.

## Ежемесячный чек-лист (10 минут)

- [ ] **Метрика → Конверсии:** сколько заявок/звонков за месяц, откуда.
- [ ] **Директ → Мастер отчётов:** расход, цена заявки, что отключить/усилить.
- [ ] **Директ → Поисковые запросы:** дочистить минус-слова.
- [ ] **Вебмастер → Страницы в поиске:** растёт ли индекс, нет ли ошибок.
- [ ] **Вебмастер → Диагностика:** исправить предупреждения.
- [ ] Обновить/добавить кейсы на сайт (свежие ответы банков) — это лучший контент для доверия.

## Частые вопросы

**«Директ не пропускает объявления»** — чаще всего из-за обещаний результата или отсутствия политики конфиденциальности. Политика теперь на сайте (`/politika-konfidencialnosti`), а формулировки в объявлениях — нейтральные (см. пакет). Также заполните реквизиты (ИНН) в подвале — для юруслуг это важно.

**«Мало заявок в первый месяц»** — это нормально для минимального бюджета. Задача месяца №1 — собрать статистику и минус-слова, а не объём. Дальше — «Оплата за конверсии» и расширение.

**«Не считаются звонки»** — проверьте, что цель `phone_click` заведена в Метрике (Часть 1.2) и что счётчик привязан к Директу (3.1).
